# Network Download Administration (NDA)

A personal IDM-style download manager: a browser extension, a local FastAPI server, and a PySide6 desktop app working together to handle direct file downloads and YouTube/Instagram/X video downloads with full quality selection.

## Features

- Segmented, multi-connection direct file downloads (pause/resume/cancel)
- Accurate file-type detection via content byte-sniffing (not just unreliable Content-Type headers) — correctly identifies real file types even from signed/tokenized URLs with no filename in the path
- YouTube, Instagram, and X video downloads via yt-dlp, with full quality ladder (144p–2160p)
- Vendored PO-Token provider (bgutil) for unlocking full YouTube resolution without manual setup
- Right-click "Download with Network Download Administration" context menu
- Floating in-page download button on video sites
- Cookie-based authentication support for private/login-gated content (optional, user-granted permission)
- Download categories (Video, Music, Pictures, Documents, Compressed, Programs, Other)
- Download history with filtering (All / Unfinished / Finished / by Category)
- System tray icon with active downloads menu
- Clipboard link monitoring (optional)
- Settings: save folder, connection count, speed limit
- Works in Chrome, Edge, Brave, Opera, Vivaldi, and Firefox

## Architecture

- **browser-extension/** — Chrome/Chromium-based extension (Manifest V3): intercepts downloads, right-click menu, popup UI
- **firefox-extension/** — Firefox-compatible build of the same extension (separate manifest for Firefox's background script requirements)
- **desktop-app/server/** — FastAPI server (127.0.0.1:5000): receives requests from the extension
- **desktop-app/gui/** — PySide6 dialogs and main window
- **desktop-app/downloader/** — download engine (direct downloads) and yt-dlp wrapper (YouTube/Instagram/X)
- **desktop-app/pot-provider/** — vendored bgutil PO-Token HTTP server (auto-started/stopped with the app)
- **desktop-app/deno/**, **desktop-app/ffmpeg/** — vendored runtime dependencies (no system-wide install needed)

## Setup

### Prerequisites
- Python 3.11+
- Google Chrome, Edge, Brave, Opera, Vivaldi, or Firefox

### Install
```bash
cd desktop-app
python -m venv venv
venv\Scripts\Activate.ps1        # Windows PowerShell
pip install -r requirements.txt
```

Place `ffmpeg.exe` inside `desktop-app/ffmpeg/` and `deno.exe` inside `desktop-app/deno/`.

Clone the PO-token provider (required for full YouTube quality support):
```bash
git clone --single-branch --branch 1.3.1 https://github.com/Brainicism/bgutil-ytdlp-pot-provider.git desktop-app/pot-provider
cd desktop-app/pot-provider/server
../../deno/deno.exe install --allow-scripts=npm:canvas --frozen
```

### Run
```bash
cd desktop-app
python main.py
```

The app automatically starts the vendored PO-Token server in the background — no separate terminal needed.

### Load the extension

**Chrome / Edge / Brave / Opera / Vivaldi:**
1. Go to `chrome://extensions` (or the equivalent for your browser)
2. Enable Developer Mode
3. Click "Load unpacked" → select the `browser-extension/` folder

**Firefox:**
1. Go to `about:debugging#/runtime/this-firefox`
2. Click "Load Temporary Add-on..."
3. Select `firefox-extension/manifest.json`
   (Note: temporary add-ons are removed when Firefox restarts — reload as needed during development)

## Known limitations

- No `.exe` packaging yet — the app currently runs via `python main.py` from a terminal
- Extension is not published to the Chrome Web Store or Firefox Add-ons — install methods above use developer/unpacked mode
- yt-dlp's support for Instagram/X can occasionally break when those platforms change their internal APIs; run `pip install -U yt-dlp` if downloads from those sites stop working
- Links generated by third-party "link generator" sites (e.g. HubCloud-style pages) may be short-lived or session-tied — download promptly after generating the link
